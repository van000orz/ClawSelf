# ClawSelf Executor（执行层调度器）

基于币安官方 square-post 技能，设计带逻辑判断的执行器：**仅当 [数据触发] 且 [未在冷却] 时** 才调用第三层内容草稿并发布；内置配额管理、多样性保护与异常熔断。

## 输入依赖

- **market_context.json**：第二层（clawself-perception）产出，含 `socialHypeTop5`、`smartMoneyTop3`（含 inflow、direction）、`topicRushRising`、`highPriority`。
- **内容层草稿**：第三层（clawself-content）按 persona + market_context 生成的贴文正文（含 Hashtags），或由调度器触发内容层生成后再发布。
- **square-post API**：币安官方发帖接口，需配置 X-Square-OpenAPI-Key。

## 重要说明：本 Skill 不会“自动配置 Cron/计划任务”

- 本文档定义的是**执行规则与判定逻辑**（阈值、冷却、配额、多样性、熔断等）。
- 要实现“全自动”，你仍需要一个**外部触发器**让执行层按周期运行（例如：Cron / Windows 任务计划 / OpenClaw 的 scheduler / 常驻进程 `setInterval` / 总控编排器）。
- 外部触发器只负责“**定期跑一次执行层**”，至于“这次到底发不发”，由本 Skill 的规则决定。

## 1. 状态监测与触发条件

### 1.1 接入第二层数据

- 每次调度周期内读取最新的 **market_context.json**（或由感知层定时更新后的路径）。
- 从 `smartMoneyTop3` 取每项的 **inflow**（美元）、**symbol**；从 `socialHypeTop5` 取排名或热度指标（若为数组则 rank 可视为 1–5 对应前五名）。

### 1.2 阈值配置（可配置项）

满足**任一**即视为「数据触发」：

| 条件类型 | 示例阈值 | 说明 |
|----------|----------|------|
| 聪明钱流入 | `inflow > 500000`（50 万美元） | 某币种 1h 聪明钱净流入超过 50 万 |
| 社交热度排名 | `hypeRank <= 10` 或 top5 内出现某 symbol | 某币进入社交热度前 5/前 10 |
| 高优先级匹配 | `highPriority.length >= 1` | 第二层已标出与人设匹配的币/话题 |

实际阈值可由用户或配置文件指定，例如：

- `INFLOW_THRESHOLD_USD = 500000`
- `HYPE_RANK_MAX = 10`（即排名 ≤10 才触发）
- `REQUIRE_HIGH_PRIORITY = true/false`（是否要求 highPriority 非空）

### 1.3 触发逻辑

**仅当**以下**同时**成立时，才进入「生成内容 + 发帖」流程：

1. **[数据触发]**：当前 market_context 满足上述阈值中至少一项。
2. **[未在冷却]**：距离上次发帖或上次调度触发的发帖已过去 ≥ `COOL_DOWN_MINUTES` 分钟（见下节）。
3. **[当日配额未满]**：当日已发帖数 < `MAX_POSTS_PER_DAY`（见下节）。
4. **[多样性通过]**：本次拟发帖的币种/话题在 4 小时内发帖次数未超限（见第 3 节）。

若不满足任一条，本周期**不调用**内容层、**不**调用 square-post，仅记录日志或状态供下一周期使用。

## 2. 配额管理

### 2.1 每日发帖上限

- **MAX_POSTS_PER_DAY = 100**
- 以自然日（0:00–24:00，建议按 UTC 或用户时区配置）统计当日已成功发帖数。
- 若当日已发帖数 ≥ 100，本日内不再触发发帖，直至次日重置计数。

### 2.2 冷却时间

- **COOL_DOWN_MINUTES = 45**
- 每次**成功发帖**后，记录发帖时间戳；在接下来的 45 分钟内，即使数据再次触发，也**不**生成新草稿、**不**再次调用 square-post。
- 跨周期调度时：若「当前时间 - 上次发帖时间」< 45 分钟，则本周期跳过发帖。

### 2.3 状态持久化建议

- 将「当日已发帖数」「上次发帖时间戳」写入本地状态文件或 KV（如 `executor_state.json` 或 OpenClaw 提供的状态存储），以便重启或跨进程后仍能正确限流。

---

## 3. 多样性保护

### 3.1 同一币种 4 小时内限次

- 对「本次拟发帖所涉主要币种」进行统计：例如从 market_context 的 `smartMoneyTop3` / `socialHypeTop5` / `highPriority` 中取本次要突出的 1–2 个 symbol。
- 规则：**同一 symbol 在任意连续 4 小时内，成功发帖次数不得超过 2 次**。
- 若本次发帖会使该 symbol 在 4 小时内达到第 3 次，则本周期**不发布**该主题，或替换为其他未超限的 symbol/话题（若内容层支持按 topic 生成）。

### 3.2 内容模板轮换

- 在 **[深度长文]**、**[犀利短评]**、**[互动提问]** 三种类型间轮换，避免连续多帖同质化。
- 实现方式示例：
  - 维护一个「上次使用的模板」状态，本次取「下一项」循环使用：sharp_short → deep_long → interactive_question → sharp_short → …
  - 或按「发帖序号 mod 3」决定本次模板。
- 调用内容层时，传入本次选定的模板类型，生成对应风格的草稿后再提交 square-post。

## 4. 异常熔断

### 4.1 触发熔断的条件

以下情况**立即停止所有自动化发帖**，并进入熔断状态：

- square-post 接口返回 **非 000000** 的 code（如 220003 API Key 不存在、220009 每日限制、20002 敏感词、30008 违规封禁等）。
- 接口返回 **风控/安全相关** 错误码（如 2000001 账号永久禁发、2000002 设备永久禁发、20022 敏感词风险等）。
- 连续 N 次（建议 N=3）请求超时或网络错误，视为不可用，触发熔断。

### 4.2 熔断后的行为

1. **停止调度**：不再根据阈值触发新帖，不再调用内容层与 square-post。
2. **推送到用户**：将错误信息、错误码、以及「已进入熔断，请人工确认」的提示，推送到用户的 **OpenClaw 终端**（或配置的通知渠道），等待人工确认。
3. **恢复方式**：仅当用户显式确认（如通过指令「恢复执行层」或「解除熔断」）后，才清除熔断状态并恢复调度；必要时需用户检查 API Key、账号状态、内容合规后再次启用。

### 4.3 错误码与熔断对照

| 建议熔断的 code | 说明 |
|-----------------|------|
| 220003, 220004  | API Key 无效或过期，需重新配置 |
| 220009          | 当日 OpenAPI 发帖已达上限 |
| 30008, 2000001, 2000002 | 账号/设备封禁，必须人工处理 |
| 20002, 20022    | 敏感词，需检查内容与策略 |
| 20020, 220011   | 空内容等，多为逻辑错误，可熔断后排查 |

## 5. 执行流程汇总

1. **读取** market_context.json、executor_state（当日计数、上次发帖时间、上次模板、熔断状态、4h 内各 symbol 发帖记录）。
2. **若处于熔断状态**：本周期仅上报状态，不执行 3–7。
3. **状态监测**：判断是否满足阈值（inflow / hype rank / highPriority）；若不满足则本周期结束。
4. **配额与冷却**：若当日发帖数 ≥ MAX_POSTS_PER_DAY 或距上次发帖 < COOL_DOWN_MINUTES，本周期结束。
5. **多样性**：确定本次要用的 symbol/话题；若该 symbol 在 4h 内已发 2 次，则跳过或换 topic；并选定本次模板（犀利短评/深度长文/互动提问）轮换。
6. **调用内容层**：传入 persona、market_context、本次模板类型，获取一篇草稿 fullText。
7. **调用 square-post**：将 fullText 作为 bodyTextOnly 提交；成功则更新「上次发帖时间」「当日计数」「该 symbol 4h 内计数」「上次模板」；失败则根据错误码决定是否**触发熔断**并推送到 OpenClaw 终端。

## 6. 配置项一览

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| MAX_POSTS_PER_DAY | 100 | 每日最大发帖数 |
| COOL_DOWN_MINUTES | 45 | 两次发帖最小间隔（分钟） |
| INFLOW_THRESHOLD_USD | 500000 | 聪明钱流入阈值（美元） |
| HYPE_RANK_MAX | 10 | 社交热度排名阈值（≤ 该值触发） |
| MAX_POSTS_PER_SYMBOL_PER_4H | 2 | 同一币种 4 小时内最多发帖次数 |
| MELTDOWN_ERROR_CODES | 见 4.3 | 触发熔断的错误码列表 |
