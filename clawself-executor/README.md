# ClawSelf Executor（执行层调度器）技能说明

## 是什么

ClawSelf 执行层的**调度逻辑**：在「何时发帖」上做条件判断，而不是无脑定时发。只有第二层数据满足阈值、且通过冷却与配额与多样性检查时，才调用第三层生成草稿并调用 square-post 发帖；遇到风控或接口异常则熔断并通知用户。

## 何时用

- 需要「按数据触发发帖」而非纯定时发帖时。
- 需要限制每日发帖数、发帖间隔、同币种频率时。
- 需要对接 square-post 且希望风控/错误时自动停发并告警时。

## 技能文件

- **SKILL.md**：完整调度逻辑（状态监测、阈值、配额与冷却、多样性保护、异常熔断、执行流程与配置项）。
- **README.md**：本说明。

## 依赖

- 第二层：market_context.json（clawself-perception）。
- 第三层：内容草稿生成（clawself-content）。
- 发帖接口：币安 square-post（见 square-post 技能文档）。

## 核心约束（默认）

- 每日最多发帖 **100** 条。
- 两次发帖间隔至少 **45** 分钟。
- 同一币种 **4 小时内** 最多 **2** 条。
- 内容类型在 **犀利短评 / 深度长文 / 互动提问** 间轮换。
- 接口错误或风控相关错误码 → **熔断**，推送到 OpenClaw 终端等人为确认后再恢复。
